Release v1.6.0-rc3 (22/1/2009)

Changes since v1.6.0-rc1:
  • Tested with gcc versions upto 4.3.2 and ROOT v5.22/00 (current production version), on Linux and MacOSX.
  • New class KVElasticScatterEvent generates events corresponding to elastic scattering of both projectile and target nuclei.
  • The previous fix for Bug Id#9 was not correct. See ROOT Forum topic here. Correct bugfix now implemented, see KVReconstructedNucleus::Copy.
  • User analysis of raw INDRA-VAMOS data (datasets INDRA_e494s and INDRA_e503) is now implemented, although for the time being no direct access to VAMOS parameters is provided. Base class for analyis is KVIVRawDataAnalyser.
  • KVDataAnalysisTask extended to allow base classes for user analysis to be in plugin libraries which are not loaded by default, even when choosing the dataset in question. See KVDataAnalysisTask::SetUserBaseClass.
  • For INDRA-VAMOS data, logfiles produced by the automatic pulser ramp programme (AUTOMATE) containing the pulser amplitudes used for each detector can now be found in the respective dataset directories ($KVROOT/KVFiles/INDRA_e494s etc.).
  • Database GUI: interface updated, now uses same KVListView display as KaliVedaGUI.
  • KVClassFactory now handles method overloading (methods with same name, different arguments) and constructors with arguments.
  • INDRA 5th campaign: data on mean values of pulser & laser for all detectors & pin diodes now integrated into database. Handled by new class KVINDRAPulserDataTree. Access to data through either KVINDRADBRun::GetMeanPulser (for a given run) or gIndraDB->GetPulserData()->GetMean (for any run in database).
  • INDRA-VAMOS: VAMOS scalers are now included in database for INDRA_e503 dataset.

Release v1.6.0-rc1 (18/12/2008)

Changes in this version:
  • Compilation tested with gcc versions upto 4.3.2 and ROOT v5.21/06 (current development version).
  • Fixed Bug Id#9 which prevented analysis of data using ROOT versions from v5.20 upwards. This was due to a change in ROOT class TRefArray, which has been 'fixed' in KVRList.
  • Visualise detectors and multidetector arrays using ROOT geometry package. See KVMultiDetArray::CreateGeoManager and KVDetector::AddToGeometry. See also new images of different INDRA configurations in class descriptions of KVINDRA and child classes.
  • Energy losses in target are automatically included by default in calculation of energy of reconstructed particles. See KVReconstructedNucleus::Calibrate.
  • Automatic CVS checking for available updates to source code now possible thanks to new KVCVSUpdateChecker class. See Configuration for how to enable.
  • BQS batch jobs at CCIN2P3: the resource 'u_sps_indra' which is severely limited (no more than 60 tokens, i.e. jobs, at the same time) is only declared when submitting a job if the working directory when the job is submitted contains "/sps/" in its path. In other words, users should not submit jobs from an /sps/ directory unless they really need to use /sps/.
  • New look KaliVedaGUI uses new layout widget KVListView: general interface for sortable lists of objects with arbitrary number of characteristics which can be displayed in columns and used to sort the list by clicking on the column headers. The user can interact with the objects in the list by double-clicking them or via their context menus.
  • All ROOT files generated from raw data now have a TTree "RawData" which simply contains a branch for every acquisition parameter and contains the values of every acquisition parameter for every event in the raw data file.
  • New global variable classes:
    • KVTensE (kinetic energy tensor)
    • KVCalorimetry (calorimetry of hot nuclei)
    • KVRelativeVelocity (calculate mean and variance of relative velocities between nuclei in event)
  • New class KVEventListMaker: generate event lists from many files containing TTrees.
  • New methods Initialize() and IsReadyForID() in KVIDTelescope. Initialize() must be overridden in child classes (i.e. specific implementations of identifications) to set IsReadyForID() flag (when it should), otherwise telescopes will not be used for identification.
  • New methods for multidetector arrays, KVMultiDetArray::PrintStatusOfIDTelescopes and KVMultiDetArray::PrintCalibStatusOfDetectors
  • New functionality in KVVarGlob global variable classes: any particle selection criteria set with SetSelection() will be applied for any global variable, if used with a KVGVList (i.e. in a KVSelector-derived analysis class)
  • Corrected systems for INDRA 5th campaign: 136Xe+124Sn 45AMeV appeared twice, with two different sets of runs
  • KVReconstructedNucleus::GetStatus(): changes name and acquires new documentation
  • KVDetector::GetCorrectedEnergy: added argument "Bool_t transmission". Default is kTRUE. Set to kFALSE for particles which stop in the detector.
  • KVDetector::FindZmin: corrected, now works for both KVChIo and KVSilicon detectors.
  • Pulse height defect (if defined) for silicon detectors integrated into calculation of energy loss. This corrects GetCorrectedEnergy() method which was not right before.
  • KVIDTelescope::CalculateParticleEnergy: improved calibration algorithm. Better treatment of 'coherency' and detectors which do not work. Introduces correction of particle energy for target losses.
  • KVDetector/KVACQParam::Fired: new options "Pall"/"Pany" (KVDetector) and "P" (KVACQParam), return kTRUE if acquisition parameter(s) has value greater than current pedestal.
  • Fixed bug in KVDBSystem::GetAtot()
  • KVTGIDFunctions: corrected divergences in functionals for total light output less than fitted pedestal value.
  • KVIDSi150CsI : greatly improved. Corrects problems with mass fits.
  • New class KVLockfile : simple interface to Linux 'lockfile' command (unlike ROOT TLockFile)
  • New methods in KVList for generating sublists from name, type, label, class, etc., also for generating list from contents of a ROOT file.
  • "New" analysis task 'raw->ident' allows to generate directly identified & calibrated data from raw data (no intermediate generation of 'recon' data files).
  • New methods in KVString: Begin(), Next(), End() allowing to loop over items in a string separated by a given delimiter character.
  • New methods AddGroup(), BelongsToGroup() in KVParticle allow more flexible sorting of particles in analysis. See example of use in updated analysis class AnalyseCamp1.

Release v1.5.5-rc7 (18/10/2006)

This update corrects several bugs and other issues:
  • it was not possible to identify all particles in "recon" events, because the KVTGID objects used to identify in Si-CsI and ChIo-CsI telescopes were not fully persistent: although the parameters of the identification functionals were stored, the function pointer needed to be reinitialised when the object was read back from a file. This has now been fixed by storing the name of the function (which has to be a member of the KVTGIDFunctions namespace) in the KVTGID object, and automatically reinitialising the function pointer of a read object in a customized streamer. "recon" data written with previous versions should be replaced.
  • new KVTGIDFunctions functional 'pichon_Z' is exactly the same as 'tassangot_Z' but uses the approximative mass formula A=2*Z rather than the EAL mass formula. This is used for the INDRA 5th campaign Si-CsI and ChIo-CsI identifications.
  • raw data files could not be read directly on remote file systems such as hpss at CCIN2P3, but had to be first copied to a local disk, taking time and disk space. Now it is possible to use the rfio protocol to read raw data files directly without an intermediate copy. Thus the procedure to open an INDRA raw data file becomes the same whatever the data repository.
  • added an "update" option to KVDataSet::GetDataBase() in order to force the regeneration of the database from (potentially changed) parameter files, rather than having to delete the DataBase.root file by hand.
  • run-sheet reader ("kvrunsheetreader") used as electronic logbook in INDRA experiments can now be used to view information for any dataset: when interface is launched, the user must first choose the dataset they wish to view.
  • identification algorithm originally implemented for ChIo-Si grids is now used as a generic identification algorithm by all Z-identification grids (KVIDZGrid). In the next version, a generic version of the CsI R-L algorithm should be implemented in KVIDZAGrid for any Z & A identifying grid.
  • the unnecessary (and unnerving) messages commonly seen at the end of raw data analysis jobs ("Not OK" and "Unknown header: ENDRUN") should have disappeared.
  • new instructions for downloading and installing - see Help or Download pages.
  • new Tutorials - a full introductory course in 4 lectures!

Release v1.5.3-rc2 UPDATED

2nd release candidate.

Several bugs corrected:

  • ChIo and Silicon energies are calculated uniquely using PG coder values. It was found that the calibration channel-volt has a discontinuity at the passage from GG to PG, which then appeared in the calibrated energy.
  • In order to improve the continuity of Z identification between different KVTGID objects, in KVTGIDManager::IdentZ we add/subtract 2 units to Zmin and Zmax. A 'hole' was seen at Z=10 for 5th campaign identification with KVIDSiCsI5, just at the Z where we change from the "GG" functional to the "PG1" functional.
  • KVIDSiCsI5 now has a lot more subcodes, in order to know which of the 2/3 functionals was used to identify (or not) the particle.
  • In KVIDSiCsI5::GetIDMapY the ad hoc gain correction of Si PG/GG coder values now takes into account the pedestal.
  • Bug in ReadUserBrute.C corrected: the UserBegin() function was called at the beginning of each new run, instead of once only at the beginning of the analysis

Release v1.5.2

CVS branch tag updates_v1_5_2.
Nouvelles classes KVTGID et KVTGIDManager qui gèrent l'identification de particules chargées en utilisant les fonctionnelles développées par Laurent Tassan-Got de l'IPN Orsay. Les fonctionnelles elles-mêmes ont été intégrées dans un namespace KVTGIDFunctions. Les ID téléscopes utilisant des identifications de cette nature ont KVTGIDManager comme une de leurs classes de base (voir e.g. KVIDSiCsI5). Cette classe permet également de visualiser les fonctionnelles d'identification sous forme de grille d'identification (KVTGIDManager::GetTGIDGrid).

D'autres nouveautés dans cette version (voir CVS Logs pour les détails): les matériaux "aluminium" et "KCl" ont été rajoutés à KVMaterial; on garde plus d'informations qu'avant sur la reconstruction des particules dans les événements écrits dans des fichiers ROOT (KVReconstructedNucleus et KVReconstructedEvent); on peut changer la couleur, le style et l'épaisseur des grilles d'identification (KVIDGrid); nouvelles méthodes GetIDMapX et GetIDMapY formalisent la façon dont les identifications se font à partir des informations fournies par les détecteurs d'un ID téléscope; amélioration des informations sur les runs INDRA 4ème campagne (rajout d'infos cibles et triggers); calibrations ChIo 5ème campagne complétées (couronnes 2-9 pour runs 6001-6979 et couronnes 10/12 pour runs 6980-fin); pour tenir compte du changement de gain des Si(300µm) sans refaire l'identification Si-CsI on modifie la valeur retournée par KVIDSiCsI5::GetIDMapY (provisoire - à contrôler); KaliVeda permet d'analyser directement des données brutes sans faire toute la reconstruction/identification - option 1. ANALYSE RAW DATA; généralisation des méthodes SetFrame de KVParticle et KVEvent, maintenant on peut définir des repères "tournés" et/ou "boostés"; méthode GetRotation de KVTenseur3 génère la rotation nécessaire pour définir le repère associé aux vecteurs propres du tenseur d'impulsion (voir FAQ).

Release v1.5.1

CVS branch tag updates_v1_5_1.
Minor update. Most changes concern database. Therefore you should delete all $KVROOT/KVFiles/KVINDRADB*.root files before using this new version.
Description of systems in database can now include details about targets (multilayered, angle to beam), see KVINDRADB::ReadSystemList. Handling of calibration peaks has been generalised to any campaign, new method KVINDRADB::GetCalibrationPeaks allows to access peaks for any detector (replaces KVChIo and KVSilicon::GetRunCalibrationPeaks). Handling of detector gains generalised to all campaigns, consequently KVDBRun5 class is obsolete.

KaliVeda release v1.4.2

This minor update concerns the KVTarget class, which now handles multi-layer targets and any arbitrary orientation of the target to the beam direction. The KVMultiDetArray::DetectParticleIn method no longer handles elastic scattering, eventually a KVScatterEvent class will provide a general way of generating elastic or inelastic scattering events which can then be 'filtered' by the detector array.
Also, for INDRA 5th campaign data, a new calibration of Silicon and ChIo detectors has been included.
This version is installed on ccali machines, or can be downloaded for local installation.
To update an existing CVS working copy, use cvs update -dP -r updates_v1_4_2
