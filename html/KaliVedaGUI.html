The KaliVedaGUI

KaliVeda



The KaliVedaGUI is a graphical interface which launches analysis tasks for INDRA data. It is launched by typing KaliVedaGUI in a terminal window. Its use is quite intuitive. We will nevertheless explain some features. Here is a snapshot of the interface.

Data Repository selection

To select a data repository, just click on the right hand side down arrow of the "Repository" combo box. Then select the data repository in the pull down menu.

 

Data Set (campaign) selection

To select a data set, just click on the right hand side down arrow of the "Data Set" combo box. Then select the data set in the pull down menu.

 

 

Analysis task selection

To select an analysis task, just click on the right hand side down arrow of the "Task" combo box. Then select the analysis task in the pull down menu. When the "recon->ident" or "ident->root" tasks are selected, the KVSelector text field and the KVDataSelector text field are set automatically and cannot be edited.

 

 

System selection

To select a system, just click on the right hand side down arrow of the "System" combo box. Then select the system in the pull down menu.

 

 

Trigger selection

To select a trigger, just click on the right hand side down arrow of the "Trigger" combo box. Then select the trigger in the pull down menu. If "All" is selected, all available runs will be displayed.

 

 

Runs selection

To select the runs to analyse, just click on the runs in the list box. The selected runs will be highlighted. You can select more than one run at the same time. You can select/deselect all runs by clicking on the "Select All"/"Deselect All" button just below the runs list box. The selected run list will appear in the field "Selected runs".

 

 

Choice of the KVSelector

Indicate in the "KVSelector" text field the name of the KVSelector you want to use for your analysis.

 

 

Choice of the KVDataSelector

Indicate in the "KVDataSelector" text field the name of the KVDataSelector you want to use for your analysis.

 

 

Choice of the number of events to read

Indicate in the "Number of events to read" text field the number of events to read. 0 means all events. The number of events can be increased or decreased by using the up or down arrows.

 

 

Selection of the user include paths and the user libraries

If needed, user include paths and user libraries can be set by clicking on the corresponding buttons. These include paths and libraries will be added to the system just before the compilation of the KVSelector.

 

 

The user libraries GUI manager looks like this:

 

 

To add a library to the system, type its full path in the text field en click on the "Add File" button. If the text field is empty when the "Add File" button is pushed, a file dialog will appear. The list of libraries which will be loaded appear in the list box. To remove some on them, just select them by clicking on them and click on the "Remove File" button. To remove all the libraries from the list, just click on the "Remove All Files" button. Once all is set, you have to close this GUI by clicking on the "Done" button. The user include paths GUI is almost identical to this GUI.

 

Selection of the running mode

Three running modes are available:

  • Interactive : the analysis is processed interactively. No more operation can be done until the analysis is completed.
  • Sequential batch : one analysis batch is launched for all the selected runs.
  • Parallel batch : one analysis batch is launched for each selected run.
Select one of these modes by clicking on the corresponding radio button.

 

 

Choice of the batch name

In case the analysis is processed in batch mode, indicate in the "Batch Name" text field the batch name. In case the "Parallel Batch" running mode is selected, a suffix containing the run number is added at the end of the batch name. If the "Auto" check box is checked, the batch name is automatically build according to the contain of the "Batch Name Format" text field. The syntax of the batch name format is the following:

  • The batch name format may contain one of these keywords:
    • $Selector                      name of the KVSelector
    • $DataSelector              name of the KVDataSelector
    • $System                       short system name
    • $Date                           current date and time
    • $User                           user's name
    • $UserClass              name of the user's class used for the analysis (including those not deriving from the KVSelector class)
  • The keywords will be replaced automatically according to the content of the different text fields and combo boxes.

Setting the batch properties

The batch properties can be set by filling the "Batch Memory", the "Batch Disk" and the "Batch Time" text fields. No white space should be in between the amount and the units for batch memory and the batch disk. The units have to be one of the followings: KB, MB or GB. The batch time can be set by using the up and down arrows of the "Batch Time" text field.

 

 

Running the analysis

Clicking on the "Procces" button launches the analysis. The ".C" and ".h" files of the the KVSelector are compiled if they exist. You will be warned if the KVSelector is not valid (missing files, it does not inherits from a KVSelector or there are compilation errors). 

 

Quit the KaliVedaGUI

Clicking on the "Quit" button closes the KaliVedaGUI window and terminates the application..
