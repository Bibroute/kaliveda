KaliVeda v1.8.2 Release Notes

KaliVeda



Here is a (far-from-complete) list of what's new in KaliVeda v1.8.2.
For more details, see here.

You should probably also read the Release Notes for v1.8.1a.

Grid Engine batch system (CCIN2P3)


Replacement of BQS by GE at CCIN2P3. v1.8.2 only works with GE (BQS became obsolete on 6/12/2011).
Change is nearly transparent for users, except:
  • memory and scratch disk request units are now 'M' instead of 'MB', 'G' instead of 'GB'
  • 'analyse_logs' works for log-files produced by 'parallel' batch jobs with 1 run per job, but no information is available on consumed scratch disk space (none is given by GE)

E613


Data for new experiment E613 (September-October 2011) can be accessed with this version. Pedestals and some other calibration parameters have been implemented.

INDRA data reduction


The 'ident->root' data reduction task has been implemented. It produces data files which include all identified and calibrated particles (including those revealed by coherency analysis, i.e. "code 6" particles).

Changes to data structures


Due to recent changes to data structures, not all informations are available when reading data files written with version 1.8.0 or 1.8.1.
In 'root' files generated from DST (INDRA campaigns 1, 2, & 4), no access to raw data parameters is possible. The files should be regenerated with v1.8.2 in order for this to be possible.

Reorganisation of analysis tasks


The "raw->ident" analysis task has been suppressed; it is obsolete as now "recon" data is independent of any changes made to detector geometry, calibrations, etc. "recon" data should be generated from raw data once and for all, and all subsequent processing done by "recon->ident" task.
The (temporary) "recon->root" task has been removed.
Data reduction schema is: 1) raw->recon; 2) recon->ident; 3) ident->root
