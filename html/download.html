Downloads



KaliVeda is available to download and build from source from our Github repository, or you can install pre-compiled Ubuntu packages for the most recent Ubuntu versions.


Source code

Source code is available from Github either in the form of compressed archives (tarballs) of released versions, or you can just clone/fork the repository in order to keep up to date with the latest developments and bugfixes using Git.

Releases
Source code for all released versions are available here. Releases can be downloaded as either a .zip or a .tar.gz archive.
Git
You can clone the git repository by doing:
$ git clone https://github.com/kaliveda-dev/kaliveda.git Cloning into 'kaliveda'... remote: Counting objects: 30161, done. remote: Total 30161 (delta 0), reused 0 (delta 0), pack-reused 30161 Receiving objects: 100% (30161/30161), 363.74 MiB | 9.01 MiB/s, done. Resolving deltas: 100% (21645/21645), done. Checking connectivity... done.
This will create a directory called 'kaliveda' containing the source code. To update the source code after the initial clone, you can do:
$ cd kaliveda $ git pull
To get the source code corresponding to a specific release, you can do:
$ git checkout -b [release-tag] [release-tag]
To see the list of all available release tag names, do:
$ git tag

Once you have downloaded the source tree, you need to Build & Install the toolkit.


Ubuntu packages

Ubuntu packages are available for either released versions or daily builds from the latest Github revision. Choose one or the other, if you add both repositories to your system there may be conflicts. Note that these packages depend on the installation of the ROOT Ubuntu packages: if they are not already installed, installing KaliVeda packages will trigger the installation of ROOT packages first. If you have "hand-built" ROOT installation(s) already on your system, this will probably lead to some weird conflicts in the future, so you should first remove any trace of ROOT from your system (other than the packaged version) before using these packages.

See here for more details on these Ubuntu packages.

Release packages
Packages for releases are available from the PPA archive ppa-kaliveda. To add this repository to your system, do the following (requires 'sudo' permissions):
$ sudo add-apt-repository ppa:kaliveda-dev/ppa-kaliveda $ sudo apt-get update
Daily builds
Packages for the latest Github version are available from the PPA archive ppa-kaliveda-daily. To add this repository to your system, do the following (requires 'sudo' permissions):
$ sudo add-apt-repository ppa:kaliveda-dev/ppa-kaliveda-daily $ sudo apt-get update
After adding one of the repositories as shown above, you can install the main KaliVeda packages like this:
$ sudo apt-get install kaliveda
There are other packages corresponding to run-time libraries for specific experiments.
