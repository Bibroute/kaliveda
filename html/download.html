Download and install from source

KaliVeda



KaliVeda latest version : KV_VERSION (KV_DATE)

There are two ways to obtain the sources to install KaliVeda on your local system: download a compressed archive (tarball) or use CVS. Only the latest version of KaliVeda is available as a tarball. With CVS, on the other hand, you can choose which version to download, or contribute to the development effort.

Download latest tarball


KaliVeda KV_VERSION source tarball (KV_DATE) : KaliVeda-KV_VERSION.tgz

After downloading, use the following command:

tar zxvf KaliVeda-KV_VERSION.tgz

This will create the directory KaliVeda-KV_VERSION.
Before installation, you need to unpack the sources:

cd KaliVeda-KV_VERSION
make unpack

Now follow the instructions in file INSTALL or here.
Details of the build system (Makefile targets & options) can be found here.

CVS access


There are two possible ways to access the sources in the CVS repository: anonymous (read-only) access, available to anybody; or full read- and write-access, only available to developers. The latter requires a login and password: contact John Frankland.

If you have any questions about using CVS, consult this excellent website.

Anonymous CVS access (read-only)


Configure your OpenSSH client as follows*:

Add the following entry to your ~/.ssh/config file:
Host anoncvs.in2p3.fr
    Port 2222
    User anoncvs
    PasswordAuthentication yes
    RSAAuthentication no
    PubkeyAuthentication no
    ForwardX11 no
    ForwardAgent no
Make sure that the following environment variables are set:
CVSROOT: anoncvs@anoncvs.in2p3.fr:/cvs/indra
CVS_RSH: ssh
*for more details, or if you do not have an OpenSSH client, see here (in French).

Download/checkout sources for any version


Once your CVS/SSH client is configured correctly, you can download a copy of the sources for any version of the code using version tags of the form: release_vX_Y_Z-rcR
    e.g. for v1.5.7-rc4, the version tag is release_v1_5_7-rc4.
See note below for versions before v1.5.7-rc4.

Use the following command to download an entirely new copy of the source:

cvs -q -z3 checkout [-r version_tag] KaliVeda

If you want to update your existing working copy to correspond to a given version, type the following command in the top-level directory of your working copy:

cvs -q -z3 update -dP -r version_tag

If you want to update your existing working copy to correspond to the most recent working version of the sources from the CVS 'HEAD' (no guarantees!):

cvs -q -z3 update -AdP

If you have any questions about using CVS, consult this excellent website.

For installation instructions see the file INSTALL in the main source directory (for old versions, you might have to deduce the installation instructions from the Makefile!!!) or here.
Details of the build system (Makefile targets & options) can be found here.

IMPORTANT: Versions before v1.5.7-rc4

Due to a change in the project structure, to obtain a correct version of KaliVeda from before v1.5.7-rc4, you must checkout separately the (obsolete) KVVarGlob and KVTrieur projects:

1. Check out the main source tree:
cvs -q -z3 checkout -r [version_tag] KaliVeda

2. Change directory to the newly-created KaliVeda directory:
cd KaliVeda

3. Check out the KVVarGlob project:
cvs -q -z3 checkout -r [version_tag]  KVVG

4. Check out the KVTrieur project:
cvs -q -z3 checkout -r [version_tag]  KVTrieur

Make sure that you give the same version_tag in each checkout !
